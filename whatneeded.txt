7. Unsigned iOS build target
   The project now defaults to:
   iossimulator-x64

   This supports unsigned simulator/build-output verification.

   Physical iPhone/iPad installation is different: Apple requires code signing to install on a real device. Since unsigned output is the goal, do not use ios-arm64 unless you later decide to configure signing.

   If ios-arm64 is selected without a CodesignKey, the project disables code signing for build-output inspection.
